### Теоретические сведения

#### Цветовая модель RGB

Цвет в модели `RGB` задается тремя компонентами:

- `R` — интенсивность красного канала
- `G` — интенсивность зеленого канала
- `B` — интенсивность синего канала

В работе изображение загружается как трехканальное и далее переводится в диапазон `[0; 1]`:

`R = R8bit / 255`, `G = G8bit / 255`, `B = B8bit / 255`

Выделение отдельного канала выполняется занулением двух остальных каналов.
Например, для красной компоненты формируется изображение вида:

`(R, 0, 0)`

Аналогично строятся изображения `(0, G, 0)` и `(0, 0, B)`.

#### Цветовая модель HSI

В модели `HSI` используются три компоненты:

- `H` — hue, цветовой тон
- `S` — saturation, насыщенность
- `I` — intensity, яркость

В скрипте используются стандартные формулы перехода `RGB -> HSI`.

Яркость:

`I = (R + G + B) / 3`

Насыщенность:

`S = 1 - min(R, G, B) / (I + eps)`

Тон вычисляется через угол `theta`:

`theta = arccos( 0.5 * ((R - G) + (R - B)) / (sqrt((R - G)^2 + (R - B)(G - B)) + eps) )`

Далее:

- если `B <= G`, то `H = theta`
- иначе `H = 2pi - theta`

Для удобства в коде тон нормируется в диапазон `[0; 1]` делением на `2pi`.

#### Инверсия яркости

После перевода изображения в `HSI` инвертируется только компонент `I`, а `H` и `S` сохраняются:

`I_inv = 1 - I`

После этого изображение переводится обратно из `HSI` в `RGB`. Такой подход позволяет изменить именно яркость, не меняя цветовой тон и насыщенность напрямую.

#### Передискретизация

Передискретизация — это изменение пространственного разрешения изображения.

В лабораторной работе рассматриваются:

- интерполяция — увеличение размера изображения
- децимация — уменьшение размера изображения
- двухпроходная передискретизация
- однопроходная передискретизация

В текущей реализации:

- растяжение выполняется собственной билинейной интерполяцией
- сжатие выполняется прореживанием отсчетов `img[::N, ::N]`

#### Билинейная интерполяция

Для каждого пикселя выходного изображения вычисляются соответствующие координаты в исходном изображении:

`y = (j + 0.5) * Hsrc / Hdst - 0.5`

`x = (i + 0.5) * Wsrc / Wdst - 0.5`

Далее выбираются четыре соседних пикселя:

- `Ia = I(y0, x0)`
- `Ib = I(y0, x1)`
- `Ic = I(y1, x0)`
- `Id = I(y1, x1)`

И итоговое значение вычисляется как взвешенная сумма:

`Inew = wa * Ia + wb * Ib + wc * Ic + wd * Id`

где веса определяются расстоянием до ближайших соседей по горизонтали и вертикали.

`I = (R + G + B) / 3`

Так как яркость — это одно число на пиксель, в результирующем изображении одно и то же значение записывается во все три канала `(I, I, I)`.

##### 2.5. Инверсия яркостной компоненты



- исходное изображение переведено в `HSI`
- яркость заменена на `I_inv = 1 - I`
- затем выполнено обратное преобразование `HSI -> RGB`

Идея шага:

Такой способ инверсии отличается от поканальной инверсии в `RGB`, потому что изменяется только яркость, а информация об оттенке и насыщенности сохраняется в компонентах `H` и `S`.

##### 3.1. Растяжение изображения в M раз

- выбран коэффициент `M = 2`

- выбран коэффициент `N = 3`
- выполнено прореживание отсчетов
- сохранялся каждый `N`-й пиксель по строкам и столбцам

В коде это реализовано как:

`img[::N, ::N, :]`

То есть новый массив строится без интерполяции, только выборкой части пикселей исходного изображения.

##### 3.3. Передискретизация в K = M / N в два прохода

1. Сначала изображение растянуто в `M = 2` раза.
2. Затем результат растяжения сжат в `N = 3` раза.

Таким образом коэффициент итогового масштабирования равен:

`K = M / N = 2 / 3`

Этот шаг показывает классическую двухпроходную схему передискретизации: сначала `upsampling`, затем `downsampling`.

##### 3.4. Передискретизация в K = M / N за один проход

- коэффициент сразу задан как `K = 2 / 3`

Размеры рассчитываются так:

- `Wnew = round(W * K)`

Для текущего изображения:

- `Wnew = 1200`
- `Hnew = 764`

Этот шаг полезен для сравнения с двухпроходным вариантом, так как оба способа имеют одинаковый итоговый размер, но могут давать немного разный визуальный результат.
